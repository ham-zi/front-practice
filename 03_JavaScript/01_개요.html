        JavaScript는 1995년도 브러우저에 동작을 넣기 위해 브렌던 아이크에 의해 만들어진 언어 (브라우저 전쟁)

        브라우저 전쟁을 거쳐 ECMAScript라는 표준으로 통일됐고 ES6 이후 현대적 언어로

        재탄생을 거쳐서 지금은 웹에서 가장 영향력 있는 언어로 자리 잡았다.


        
        스크립트 언어란?
        애플리케이션의 제어를 사용자의 요구에 맞게 수행할 수 있도록
        빠르게 배우고 짧은 소스코드로 상호작용이 가능한 언어들


        
        자바스크립트만의 특징은?
        웝에서 일반적으로 사용되는 스크립트 언어다.
        현시대 웹에서 화면을 만들어내는 기반기술로 사용되는것이 가장 큰 특징
        (Angular, Vue, React)


        React(Meta) Vs Vue(쉬움)
        정부에서 전자정보프레임워크를 쓰는데, 정부에서 규격을 정해줌
        
        
        컴파일 : 컴파일러가 컴파일 단계를 거쳐서 실행하는 방식
        인터프리터 : 필요할 때 마다 한 줄 한 줄 실행하는 방식 
        
        JIT 컴파일러 : 자주 실행되는 "핫 코드"를 기계어로 재 컴파일 (자바,자바스크립트 공통)
        자바, 자바스크립트 => 컴파일 + 인터프리터






        -  자바스크립트
        1. 함수를 일급 객체로 취급함(함수를 인자로 전달할 수 있음)
        2. 문법이 쉽다 (접근제한자 개념이 없다, 함수 반환형이 없다, 변수타입 지정X(타입추론))
        3. 계속 발전하고 피드백이 빠름
        4. 확장성이 좋아서 배워두면 여기저기 사용이 가능하다.


        
        자바스크립트 소스코드 작성 위치에 따른 종류
        1. inline : 요소 속성에 간단한 자바스크립트 코드를 작성
        2.internal : HTML 문서 내에 script 태그에 코드를 작성
        3.external : 코드를 별로의 .js 파일로 작성해서 읽어오는 방법
        (모듈로 만들어서 export하고 쓰는 쪽에서 import해서 불러오는 방법도 있음)

        internal 방식
        HTML문서에 script라는 태그 영역을 만든 뒤
        함수 단위로 소스코드를 작성하고
        특정 요소에 이벤트가 발생했을 때 해당 함수를 호출하는 방식

        웹브라우저에서
        렌더링엔진 자바스크립트엔진이 핵심

        요청해서 html받으면 
        파싱 ->   html dom모델트리를 만듦
             ->    css 모델트리를 만듦
        합치면 -> 렌더트리를 만듦
        이게 렌더링 엔진이 하는 일이다.

        브라우저는 두가지 엔진을 동시에 실행할 수 없기 떄문에
        중간에 script를 만나면 파싱을하다가 멈추고
        자바스크립트엔진이 구동한다

        동작하는 자바스크립트 코드가 많으면 화면 로딩시간이 길어진다.
        
        자바스크립트는 돔을 마음대로 조작할 수 있음
        -> 주 사용처임
