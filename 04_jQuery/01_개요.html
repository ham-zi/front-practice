jQuery 란?

궁금궁금 
        기존 자바스크립트만으로(옛날) 브라우저 마다 각기 다른 코드를 작성해서
        브라우저를 조작해야하는 어려움이 있었다.
        '존 레식'은 이런 문제를 해결하기 위해 JavaScript라는 언어를 이용해서
        함수 모음집을 만들어서 배포했다.
        이것이 jQuery라는 라이브러리이다.

        JavaScript의 함수를 많이 모아 놓은 파일(라이브러리)

        - 사람들은 대체 왜 그렇게 jQuery에 열광 했을까?
        1. DOM요소와 관련된 스크립트 코드를 아주 쉽게 구현할 수 있다.
        2. AJAX(비동기식 통신), 이벤트 처리를 아주 쉽게 구현할 수 있다.
        3. jQuery와 관련된 확장 플러그인, Open API등이 많이 지원된다. 


        => 조금 쓰고 많이 할 수 있다.

        단점 => 바닐라 자바스크립트 대비 속도가 너무 느리다.
        => 신규개발에서는 사용되지 않는다.

jQuery 사용하기

        jQuery를 사용해서 DOM을 다룰 예정
        jQuery코드가 문서상 모든 요소들이 전부 다 로드 된 후에
        실행할 수 있게 만들어 줘야함 => window.onload


        onload : 페이지가 전부 다 로딩되고 나면 자동으로 딱 한번 수행
        
        해당 HTML의 요소들도 전부 다 로딩되고 연동되어 있는 외부 문서들도
        전부 다 로딩된 후에 onload속성에 대입한 이벤트 핸들러가 수행되는
        특징을 가짐
    

        -jQuery에서는 해당 내용을 세 가지 구문을 제공하여 해결한다.

        1) jQuery(document)함수 호출
        jQuery(document).ready(function(){
            실행하고 싶은 내용;
        });

        2) $ : jQuery키워드를 나타냄 (단순화)
        $(document).ready(function(){
            실행하고 싶은 내용;
        });

        3) 더욱 단순화
        $(function(){
            실행하고 싶은 내용;
        });

        jQuery에서 해당 function은 HTML문서 안에 있는 DOM요소들이
        모두 로딩되면 바로 실행한다.
        여러 번 기술 가능, 횟수 제한 없음.
